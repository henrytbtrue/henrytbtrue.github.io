题解 CF1174E Ehab and the Expected GCD Problem

发表于 2021-06-12 分类于 题解 阅读次数： Valine： 本文字数： 2.8k 阅读时长 ≈ 3 分钟

毒瘤 dp 题。

首先显然可以莽一波结论。

可以发现，当排列的第一个数 a1=∏pici,pi is primea_1=\prod p_i^{c_i},p_i\ \text{is prime}a1​=∏pici​​,pi​ is prime 时，题目中所求排列的权值最大值为 ∑ci\sum c_i∑ci​。

这是因为在第二个数及以后，前缀 gcd⁡\gcdgcd 只会减小，不会增加，而且最好的情况是每次减小一个质因子。可以证明只要有一个确定的第一个数，该最好情况一定可以实现。

那么我们可以考虑找出使 ∑ci\sum c_i∑ci​ 最大的一个数 a1a_1a1​。

再来猜一波：这个 a1a_1a1​ 的可能性不会太多，它的质因子得小。因为如果有比较大的质因子的话，它的指数 cic_ici​ 就会被压榨，会比质因子更小的情况劣。

那么质因子得有多小呢？我们先猜：这个 a1=2xa_1=2^xa1​=2x。

这种情况确实很优，但不久后我们还会发现，2x⋅3y2^x\cdot 3^y2x⋅3y 似乎也很可以。

但是到 2x⋅322^x\cdot 3^22x⋅32 之后就爆炸了，因为有一个比它小但质因数分解后的指数和比它大的数 2x+32^{x+3}2x+3。

因此，质因子 333 的次数不能大于 111。

那么能不能有比 333 大的质因子呢？显然不能。因为 5>225>2^25>22，用了 555 及以上的不如用一堆 222 来代替。

那么我们可以得出结论：a1a_1a1​ 只能等于 222 的若干次方，或者 222 的若干次方乘上 333。

那么就可以愉快设 dp 状态了：令 fi,j,kf_{i,j,k}fi,j,k​ 表示到了该排列的第 iii 个数，当前前缀 gcd⁡=2j⋅3k\gcd=2^j\cdot 3^kgcd=2j⋅3k 时，有多少种情况。

转移可以分三类讨论：

  • 若加入一个数后 gcd⁡\gcdgcd 不变，那么 fi+1,j,kf_{i+1,j,k}fi+1,j,k​ 需要加上 fi,j,k⋅(⌊n2j⋅3k⌋−i)f_{i,j,k}\cdot\left(\left\lfloor\frac{n}{2^j\cdot 3^k}\right\rfloor-i\right)fi,j,k​⋅(⌊2j⋅3kn​⌋−i)。因为 gcd⁡\gcdgcd 不变，那么加入的数一定是当前 gcd⁡\gcdgcd 的倍数，需要注意前面的所有数一定都是当前 gcd⁡\gcdgcd 的倍数，不可以选已经选过的数，于是情况数需要 −i-i−i。
  • 若加入一个数后 gcd⁡\gcdgcd 变少了 222 倍，那么 fi+1,j−1,kf_{i+1,j-1,k}fi+1,j−1,k​ 需要加上 fi,j,k⋅(⌊n2j−1⋅3k⌋−⌊n2j⋅3k⌋)f_{i,j,k}\cdot\left(\left\lfloor\frac{n}{2^{j-1}\cdot 3^k}\right\rfloor-\left\lfloor\frac{n}{2^{j}\cdot 3^k}\right\rfloor\right)fi,j,k​⋅(⌊2j−1⋅3kn​⌋−⌊2j⋅3kn​⌋)。由于 gcd⁡\gcdgcd 变少两倍，那么新加入的数一定得是 gcd⁡2\frac{\gcd}{2}2gcd​ 的倍数，但它不能是 gcd⁡\gcdgcd 本身的倍数，因为这样的话 gcd⁡\gcdgcd 就不会改变了。而在 gcd⁡2\frac{\gcd}{2}2gcd​ 的倍数和 gcd⁡\gcdgcd 本身的倍数中，都考虑了一遍已经加入过的数，抵消掉了，因此不会加入重复的数。
  • 若加入一个数后 gcd⁡\gcdgcd 变少 333 被，那么 fi+1,j,k−1f_{i+1,j,k-1}fi+1,j,k−1​ 需要加上 fi,j,k⋅(⌊n2j⋅3k−1⌋−⌊n2j⋅3k⌋)f_{i,j,k}\cdot\left(\left\lfloor\frac{n}{2^{j}\cdot 3^{k-1}}\right\rfloor-\left\lfloor\frac{n}{2^{j}\cdot 3^k}\right\rfloor\right)fi,j,k​⋅(⌊2j⋅3k−1n​⌋−⌊2j⋅3kn​⌋)。由于 gcd⁡\gcdgcd 变少两倍，那么新加入的数一定得是 gcd⁡3\frac{\gcd}{3}3gcd​ 的倍数，但它不能是 gcd⁡\gcdgcd 本身的倍数。类似地，在 gcd⁡3\frac{\gcd}{3}3gcd​ 的倍数和 gcd⁡\gcdgcd 本身的倍数中，也抵消掉了重复的数。因此这么干是对的。

代码实现上因为 333 只有可能是 000 次方或 111 次方，可以用三目运算符判断更加方便。
